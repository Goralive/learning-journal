# TDD with Spring Boot Done Right

## Why?

1. It's fun
2. Change the mindset of how deliver features
3. How you know what to implement if you can't test it?
4. Proper architecture and design
5. Easy to change the code more confidence. In long term more productive
6. Easier to update libraries no manual tests
7. Living documentation

## Excuses

- To slow with TDD. In long term will be faster
- Don't care about testing
- We don't know how to test
- This feature can't implement with TDD
